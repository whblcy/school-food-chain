## 数据库设计

### 核心表

- `users` - 用户表（支持多租户）
- `organizations` - 组织架构（集团/校区/食堂）
- `suppliers` - 供应商表（含资质、评级）
- `ingredients` - 食材表
- `categories` - 分类表
- `purchase_orders` - 采购订单
- `stock_in` - 入库记录
- `stock_out` - 出库记录
- `inventory_checks` - 盘点记录
- `trace_records` - 追溯记录
- `audit_logs` - 审计日志

### 多租户设计

每个组织（organization）独立隔离数据，通过 `org_id` 字段实现行级安全。

## API 设计

遵循 RESTful 规范，使用 JWT 认证。

```
/api/v1/auth/*          # 认证
/api/v1/users/*         # 用户管理
/api/v1/orgs/*          # 组织架构
/api/v1/suppliers/*     # 供应商
/api/v1/ingredients/*   # 食材
/api/v1/stock/*         # 库存
/api/v1/finance/*       # 财务
/api/v1/reports/*       # 报表
/api/v1/trace/*         # 追溯
/api/v1/audit/*         # 审计
```

## 部署架构

```
Docker Compose:
  - nginx (反向代理)
  - backend (FastAPI, 可水平扩展)
  - postgres (主数据库)
  - redis (缓存)
  - minio (对象存储)
  - celery-worker (异步任务)
  - celery-beat (定时任务)
```
